## 1. 快速开始

### 1.1 环境要求

- 任意现代浏览器（Chrome / Firefox / Safari / Edge 最新版本）
- 无需安装、无需注册、无需后端服务器

### 1.2 打开方式

**方式一：直接打开（推荐）**

1. 下载项目中的 `index.html` 文件
2. 双击文件，浏览器自动打开

```
Windows:   双击 index.html
macOS:     双击 index.html
Linux:     xdg-open index.html
```

**方式二：克隆仓库**

```bash
git clone https://github.com/CarlosZhou6767/Newsletter-Builder.git
cd Newsletter-Builder
# 双击打开 index.html
```

打开页面后，会看到欢迎弹窗：

## 3. 三种工作模式

### 3.1 代码模式

适合开发者手写 HTML 进行精细化控制。

**界面布局**：左右分栏 — 左侧代码编辑器、右侧实时预览 iframe

```
┌──────────────────────┬─────────────────────────────┐
│  代码编辑器            │  实时预览                    │
│                      │                             │
│  <section>           │  ┌─────────────────────┐    │
│    <h1>标题</h1>     │  │                     │    │
│    <p>内容</p>       │  │  预览效果实时展示    │    │
│  </section>          │  │                     │    │
│                      │  └─────────────────────┘    │
│  [运行] [清空] [复制] [格式化]│                     │
└──────────────────────┴─────────────────────────────┘
```

**操作步骤**：

1. 点击顶部 **"代码"** 按钮进入代码模式
2. 在左侧编辑器输入或粘贴 HTML 代码
3. 点击 **"运行"**（或 `Ctrl+Enter`）刷新预览
4. 右侧 iframe 实时显示渲染效果
5. 可使用 **"格式化"** 美化代码，**"复制"** 复制到剪贴板
6. 底部状态栏显示字符数和行数

**代码编辑器特性**：
- Tab 键支持缩进（2 空格）
- 语法高亮
- 实时错误控制台（底部可展开）

### 3.2 设计模式

适合非技术用户通过可视化界面构建内容。**这是最常用的模式**，三栏布局：

- **左侧**：组件库面板（4 个标签页）
- **中间**：画布区域，组件按需排列
- **右侧**：属性面板（选中组件后显示）

**操作步骤**：

1. 点击顶部 **"设计"** 按钮进入设计模式
2. 从左侧助手面板的 **"组件"** 标签页点击插入组件
3. 在画布上点击组件选中，右侧属性面板编辑内容
4. 拖拽组件手柄调整顺序
5. 切换到 **"样式"** 标签页调整配色和排版
6. 切换到 **"检查"** 标签页运行合规检查

### 3.3 预览模式

模拟最终页面效果，全面检查呈现质量。

```
┌──────────────────────────────────────────────────┐
│  [桌面] [手机]  [← 返回设计]                      │
│                                                  │
│  ┌──────────────────────────────────────────┐    │
│  │                                          │    │
│  │          最终效果预览                     │    │
│  │                                          │    │
│  │  （与导出版本完全一致）                   │    │
│  │                                          │    │
│  └──────────────────────────────────────────┘    │
└──────────────────────────────────────────────────┘
```

**操作步骤**：

1. 点击顶部 **"预览"** 按钮进入预览模式
2. 点击 **"桌面"** / **"手机"** 切换设备宽度
3. 查看响应式效果
4. 点击 **"全屏"** 按钮进入全屏预览
5. 按 `Escape` 退出全屏

## 4. 设计模式详解

### 4.1 组件面板

**打开方式**：设计模式 → 左侧点击 **"组件"** 标签页

**内置区块列表**（共 19 种）：

| 组件类型 | 说明 | 适用场景 |
|----------|------|----------|
| `heading` | 标题 | 文章标题、栏目名称 |
| `text` | 文本段落 | 正文内容 |
| `image` | 单张图片 | 封面图、插图 |
| `image-text` | 图文混排 | 左图右文或右图左文 |
| `cta` | 按钮 | 引导点击、征稿链接 |
| `divider` | 分隔线 | 内容分区 |
| `spacer` | 空白间距 | 调整间距 |
| `list` | 列表 | 条目展示 |
| `quote` | 引用 | 名人名言、摘要 |
| `callout` | 提示框 | 重要通知、注意事项 |
| `gallery` | 图片集 | 多图展示 |
| `author` | 作者信息 | 作者名、单位、头像 |
| `journal-cover` | 期刊封面 | 期刊封面展示 |
| `toc` | 目录 | 文章目录 |
| `paper-list` | 论文列表 | 多篇论文条目 |
| `paper-card` | 论文卡片 | 单篇论文展示 |
| `event-card` | 事件卡片 | 会议、征稿截止日期 |
| `banner` | 横幅 | 促销、通知横幅 |
| `footer` | 页脚 | 版权信息、社交链接 |

**操作步骤**：

1. 在组件面板中找到需要的组件
2. **搜索过滤**（V0.2.0 新增）：在搜索框中输入关键词，实时过滤组件列表，支持按名称搜索
3. 点击组件卡片 → 自动添加到画布末尾
4. 在画布上点击组件 → 右侧属性面板出现
5. 编辑属性后实时生效

### 4.2 画布编辑

**组件选中**：点击画布上的组件，边框高亮 + 右侧属性面板出现

**组件排序**：有三种方式

- **拖拽排序**：按住组件左侧的 `⋮⋮` 手柄，上下拖拽
- **按钮操作**：在右侧属性面板点击 [上移] / [下移]
- **拖拽到末尾**：拖拽组件到画布底部空白区域

**组件操作**（右侧属性面板上方）：

| 按钮 | 快捷键 | 说明 |
|------|--------|------|
| 上移 | — | 向上移动一位 |
| 下移 | — | 向下移动一位 |
| 复制 | `Ctrl+D` | 深拷贝选中组件 |
| 删除 | `Delete` | 删除选中组件 |

**保存为自定义区块**：点击组件上的 ☆ 按钮，保存后可在组件面板的"我的区块"中复用

**剪贴板复制/粘贴**（V0.2.0 新增）：

1. 选中画布上的组件
2. 按 `Ctrl+C` 复制组件到剪贴板
3. 选中目标位置（或最后一个组件），按 `Ctrl+V` 粘贴
4. 粘贴的组件自动插入到选中位置之后

### 4.3 模板面板

**打开方式**：设计模式 → 左侧点击 **"模板"** 标签页

**内置模板**（共 7 个）：

| 模板 | 类型 | 说明 |
|------|------|------|
| Template 1 | 精选期刊 | 侧边栏精选，适合期刊 Highlights |
| Template 2 | 封面+图文 | 期刊封面 + 图文论文，适合最新录用 |
| Template 3.1 | 封面+列表 | 期刊封面 + 论文列表，适合目录页 |
| Template 3.2 | 封面+摘要 | 期刊封面 + Research Digest |
| Template 4 | 会议横幅 | 会议横幅 + 图文论文，适合征稿 |
| Template 5.1 | 会议列表 | 会议横幅 + 论文列表 |
| Template 5.2 | 会议摘要 | 会议横幅 + Research Digest |

**加载模板步骤**：

1. 在模板面板找到目标模板卡片
2. 点击卡片 → 自动加载（清空当前画布）
3. 系统自动切换到设计模式
4. 加载完成后自动保存历史版本

**自定义模板**：

1. 设计好内容后，点击 **"保存当前画布为模板"**
2. 输入模板名称，确定后存入 localStorage
3. 后续可在"我的模板"区域加载或删除

## 5. 组件操作指南

### 5.1 所有组件共有的操作

每个组件在画布上都有统一的操作栏：

```
┌─────────────────────────────────────────────────┐
│  [⋮⋮]  组件内容区域...              [☆] [⧉] [✕] │
└─────────────────────────────────────────────────┘
```

| 图标 | 操作 | 说明 |
|------|------|------|
| `⋮⋮` | 拖拽手柄 | 按住拖拽调整顺序 |
| `☆` | 保存为区块 | 保存为自定义组件 |
| `⧉` | 复制 | 复制该组件 |
| `✕` | 删除 | 删除该组件 |

### 5.2 各组件属性编辑说明

选中组件后，右侧属性面板会显示对应的编辑项。以下为各组件可编辑的属性：

| 组件类型 | 可编辑属性 | 说明 |
|----------|------------|------|
| `heading` | 文字内容、字号级别(H1/H2/H3)、对齐方式、文字颜色 | 标题文本 |
| `text` | 文字内容、文字颜色 | 段落文本 |
| `image` | 图片 URL、替代文字、链接 URL | 支持远程图片 |
| `image-text` | 图片 URL、替代文字、标题、内容、图片位置(左/右) | 图文混排 |
| `cta` | 按钮文字、链接 URL、按钮颜色 | 支持按钮颜色自定义 |
| `divider` | 线条颜色、线条样式 | 分隔线 |
| `spacer` | 高度 | 空白间距 |
| `list` | 列表项、列表样式 | 支持有序/无序列表 |
| `quote` | 引用文字、来源 | 引用块 |
| `callout` | 提示文字、背景色 | 带背景色的提示框 |
| `gallery` | 图片 URL 列表 | 多图展示 |
| `author` | 作者名、单位、头像 URL | 作者信息 |
| `journal-cover` | 封面 URL、期刊名 | 期刊封面 |
| `toc` | 标题列表 | 目录 |
| `paper-list` | 论文条目列表 | 论文列表 |
| `paper-card` | 标题、作者、摘要、链接 | 单篇论文卡片 |
| `event-card` | 事件名、日期、描述 | 事件卡片 |
| `banner` | 文字、背景色 | 横幅 |
| `footer` | 版权文字、社交链接 | 页脚 |

### 5.3 拖拽排序详细操作

**画布内排序**：

1. 鼠标悬停在组件上，左侧显示 `⋮⋮` 拖拽手柄
2. 按住 `⋮⋮` 并上下拖动
3. 拖拽过程中会出现蓝色指示线，表示放置位置
4. 松开鼠标确认放置

**从组件库插入**：

1. 在组件标签页，按住组件卡片
2. 拖拽到画布目标位置（蓝色指示线显示）
3. 松开鼠标插入

## 6. 样式编辑指南

### 6.1 颜色设置

**打开方式**：设计模式 → 左侧 **"样式"** 标签页 → 颜色区域

**主题色设置**：

| 色值 | 说明 | 默认值 |
|------|------|--------|
| 主色 | 按钮、链接等强调色 | `#667eea` |
| 背景色 | 画布背景色 | `#ffffff` |
| 文字色 | 正文默认文字色 | `#1f2937` |

**操作步骤**：

1. 点击颜色拾取器或直接输入 Hex 值（如 `#667eea`）
2. 画布实时更新配色
3. 修改主色后，预设配色和智能配色会基于新主色重新生成

**预设配色**（12 套一键切换）：

| 配色名 | 主色 | 适用场景 |
|--------|------|----------|
| 科技蓝 | `#667eea` | 学术期刊、科技类 |
| 电商红 | `#e74c3c` | 促销、活动 |
| 教育绿 | `#27ae60` | 教育、课程 |
| 节日橙 | `#f39c12` | 节日、活动 |
| SaaS蓝 | `#3498db` | 产品更新 |
| 创意粉 | `#e74c8a` | 创意、艺术 |
| 靛蓝 | `#4361ee` | 正式、专业 |
| 青绿 | `#2ec4b6` | 清新、自然 |
| 橙色 | `#f77f00` | 活力、积极 |
| 紫罗兰 | `#7c3aed` | 优雅、高端 |
| 青色 | `#06b6d4` | 科技、现代 |
| 草绿 | `#84cc16` | 环保、自然 |

**操作步骤**：点击任一色块 → 一键应用

**智能配色**（基于当前主色自动生成）：

| 模式 | 说明 |
|------|------|
| 类比色 | 生成相邻色相（±30°）的 3 色方案 |
| 互补色 | 生成对角色相（180°）的 3 色方案 |
| 三角色 | 生成三等分色相（120°）的 3 色方案 |

**操作步骤**：点击 [类比色] / [互补色] / [三角色] → 自动生成并应用

### 6.2 字体设置

**字体栈**：

| 选项 | 字体栈 |
|------|--------|
| System | `-apple-system, 'Segoe UI', Roboto, sans-serif` |
| 苹方 | `'PingFang SC', 'Microsoft YaHei', sans-serif` |
| 微软雅黑 | `'Microsoft YaHei', 'PingFang SC', sans-serif` |
| 衬线 | `Georgia, 'Times New Roman', serif` |
| 等宽 | `Consolas, 'Courier New', monospace` |

**标题层级**：H1 / H2 / H3 字号可分别通过滑块调整（16-48px）

**正文排版**：

| 参数 | 范围 | 默认值 |
|------|------|--------|
| 字重 | 300-700 | 400 |
| 字号 | 12-20px | 15px |
| 行高 | 1.4-2.0 | 1.7 |

### 6.3 间距设置

## 8. 检查与合规

### 8.1 暗色网页预览

**用途**：预览页面在暗色模式下的显示效果，确保 CTA 按钮、边框、文字在暗色模式下仍然清晰。

**操作步骤**：

1. 切换到 **"检查"** 标签页
2. 勾选 **"暗色模式预览"** 复选框
3. 画布立即切换为暗色模式（深色背景 + 浅色文字）
4. 检查每个组件在暗色下的显示效果
5. 取消勾选回到正常模式
6. 导出 HTML 时自动注入暗色适配 CSS

### 8.2 WCAG-AA 无障碍审计

**用途**：检查所有组件的文字与背景对比度是否符合 WCAG 2.1 AA 标准。

**评级标准**：

| 类型 | AA 要求 | AAA 要求 |
|------|---------|----------|
| 正文（< 18px 且非粗体） | ≥ 4.5:1 | ≥ 7:1 |
| 大字（≥ 18px 或 ≥ 14px 粗体） | ≥ 3:1 | ≥ 4.5:1 |

**操作步骤**：

1. 确保在明色模式下（编辑器默认）
2. 切换到 **"检查"** 标签页
3. 点击 **"WCAG-AA 审计"** 按钮
4. 下方显示审计报告：
   - 顶部汇总：检查项数、通过数、失败数
   - **AA 不达标**（红色）：必须修复，正文 < 4.5:1
   - **AAA 不达标**（橙色）：建议修复，正文 < 7:1

**修复建议示例**：

```
image-text · div "Interaction Regularity..." 对比度 1.89:1 / 需要 3:1（大字）
→ 建议：把文字色调至 #6C7989 可达 4.5:1
```

**修复流程**：

1. 查看报告中的组件类型（如 `image-text`、`heading`）
2. 回到画布选中该组件
3. 在右侧属性面板修改文字颜色
4. 再次点击审计按钮复检

### 8.3 发布前检查

**用途**：在导出前全面检查页面的质量。

**检查项**：

| 检查项 | 说明 |
|--------|------|
| 图片完整性 | 检查是否所有图片都有有效的 src |
| 链接空值 | 检查是否所有链接都有有效的 href |
| Alt 属性 | 检查是否所有图片都有替代文字 |
| 移动端适配 | 检查是否包含响应式 viewport meta |
| 对比度合规 | 检查主题色与背景色的对比度 |

**操作步骤**：

1. 切换到 **"检查"** 标签页
2. 点击 **"运行检查"** 按钮
3. 检查列表显示每项的通过/失败状态
4. 根据失败项调整设计

### 8.4 版本历史时间线

**用途**：查看和恢复历史编辑版本，所有快照持久化在浏览器 IndexedDB 中（最多 50 帧）。

**操作步骤**：

**查看历史**：

1. 切换到 **"检查"** 标签页
2. 滚动到底部 **"版本历史"** 区域
3. 显示垂直时间线列表，每行包含：
   - 左侧：SVG 缩略图（组件结构示意）

   - 右侧：删除按钮（鼠标悬停显示）

**恢复版本**：

## 11. 常见问题

### Q1: 数据存储在哪里？会丢失吗？

所有数据存储在浏览器本地：
- 设计内容、自定义模板、自定义区块 → `localStorage`（~5MB，清缓存会丢失）
- 版本历史帧 → `IndexedDB`（独立配额几十 MB）
- **建议定期用"导出 HTML"做异地备份**

### Q2: 能跨设备同步数据吗？

**不能**。localStorage 和 IndexedDB 均为浏览器本地存储，不同设备、不同浏览器各自独立。跨设备同步需要额外后端服务。

### Q3: 暗色预览切换后编辑器变暗了？

不会。暗色预览只影响画布区域（`.canvas-frame`），编辑器外壳由独立的 `data-theme` 控制。如异常请刷新页面。

### Q4: 审计报告出现大量 `unknown · div "⋮"` 失败项？

那是组件内部装饰性元素（拖拽手柄 `⋮`），导出 HTML 时不会出现。重点关注带内容的组件（`text`、`image-text`、`heading`），忽略 `div "⋮"` 这类。

### Q5: 版本历史里找不到刚才的版本？

可能原因：
1. 只有调用 `saveHistory` 的操作才会入帧（如撤销/重做、保存），纯拖拽不触发
2. 超过 50 帧后最旧帧被自动淘汰
3. IndexedDB 配额错误（检查 DevTools 控制台）

### Q6: 支持导入已有的 HTML 吗？

代码模式下可直接粘贴 HTML 编辑。设计模式暂不支持从 HTML 反向解析。

### Q7: 如何换电脑继续编辑？（V0.2.0 新增）

使用 JSON 导入/导出功能：
1. 在当前电脑上点击工具栏 **"JSON"** 按钮，导出 `journal-project.json`
2. 将文件传输到另一台电脑（U盘/云盘/网盘）
3. 在新电脑上打开 期刊网页构建工具，点击 **"导入"** 按钮选择该文件
4. 所有组件、主题、字体和间距设置完整恢复

### Q8: 如何完全重置所有数据？

打开浏览器 DevTools → Application → 分别清除：
- `localStorage` 中 `newsletter-` 开头的所有 key
- IndexedDB 中 `newsletter-history-db` 数据库
